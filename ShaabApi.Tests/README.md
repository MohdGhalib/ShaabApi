# ShaabApi.Tests — اختبارات الأعطال الحرجة

اختبارات تحمي إصلاحات 2026-06-07 من العودة (regression). مشروع منفصل لا يدخل في بناء/نشر التطبيق الرئيسي (مستبعد في `ShaabApi.csproj`).

## التشغيل

**اختبارات الخادم (.NET / xUnit + SQLite في الذاكرة):**
```bash
dotnet test ShaabApi.Tests/ShaabApi.Tests.csproj
```

**اختبارات الواجهة (الدوال النقية / Node):**
```bash
node --test ShaabApi.Tests/js/*.test.js
```

## ما الذي تغطّيه

### الخادم — `PerRecordSyncTests.cs`
حارس تراجع التسليم في `PerRecordSyncService`:
- مزامنة blob قديمة **لا** تُرجِع منتسية "تم التسليم" لحالة أقدم.
- الإدخال الجديد من الـ blob يُضاف بشكل طبيعي.
- التقدّم الشرعي (قيد الانتظار → تم التسليم) يُطبَّق.
- سجل التسليم تُحفظ كل حقوله (لا يُدهَس deliveredBy).

### الواجهة — `js/pure-functions.test.js`
- `_canonRec` (المقارنة القانونية): `offerName: ''` يساوي `null`، المفتاح المفقود يساوي `null`، اختلاف الترتيب لا يُحدث فرقاً، واختلاف المحتوى الحقيقي يُكتشف — يمنع حلقة الإرسال/409 اللانهائية.
- `_montasiaPhotoSrc`: رابط جديد (`photoUrl`) أو base64 قديم، مع الأولوية الصحيحة.
