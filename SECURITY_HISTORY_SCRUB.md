# 🧹 مسح الأسرار المسرّبة من تاريخ git

## ⚠️ الأهم أولاً: التدوير يُبطل الأسرار القديمة
بعد تشغيل `setup-local-server.ps1` بكلمات مرور **جديدة**، تصبح القيم القديمة
(`090999797269`, `0785110515`, الهاشات، مفاتيح JWT القديمة) **عديمة الفائدة**.
لذا مسح التاريخ يصبح إجراء نظافة لا ضرورة عاجلة. **دوّر أولاً.**

## لماذا لا أُنفّذ المسح تلقائياً؟
إعادة كتابة تاريخ git عملية **مدمّرة لا رجعة فيها** (تغيّر كل الـ hashes)، وهذا
المستودع كبير (تاريخ node_modules). كما أن `0785110515` هو **رقم هاتف تواصل
حالي مشروع** (في `lockdown.js`) — فاستبداله الأعمى يُفسد الكود الحالي.

## الخيار الموصى به للسيرفر الداخلي: بداية تاريخ نظيفة
أبسط وأأمن طريقة لإزالة كل الأسرار القديمة دفعة واحدة (تفقد سجل الالتزامات):
```bash
# 1) خذ نسخة احتياطية أولاً
cp -r .git ../git-backup

# 2) ابدأ تاريخاً جديداً من الحالة الحالية (النظيفة)
git checkout --orphan clean-main
git add -A
git commit -m "chore: clean history start (secrets rotated & removed)"
git branch -D main
git branch -m main
```

## الخيار البديل: استبدال نصوص محدّدة عبر git-filter-repo
يُبقي السجل لكن يستبدل الأسرار بـ `***REMOVED***` في كل الالتزامات.
> لا تُدرج `0785110515` (رقم هاتف حالي). نُدرج فقط ما لم يعد مستخدَماً.
```bash
pip install git-filter-repo        # لمرة واحدة
# انشئ ملف secrets-to-scrub.txt (انظر القالب المرفق) ثم:
git filter-repo --replace-text secrets-to-scrub.txt --force
```

ملف القالب `secrets-to-scrub.txt` مرفق في هذا المجلد — راجِعه وعدّله قبل التشغيل.

## بعد المسح
- أعد إضافة الـ remote إن لزم وادفع بـ `--force` (نسّق مع أي متعاون أولاً).
- احذف النسخة الاحتياطية `../git-backup` بعد التأكد.
