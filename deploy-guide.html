0) نظرة عامة — ماذا تركّب بالضبط؟

النظام بسيط في جوهره: تطبيق واحد مكتوب بلغة .NET 8 (ASP.NET Core) اسمه ShaabApi. هذا التطبيق يقوم بدورين في آنٍ واحد:

  • يخدم واجهة الموقع (صفحات HTML/JavaScript التي يراها الموظف في المتصفّح).
  • يخدم الـ API (يستقبل ويحفظ البيانات).

ويحفظ كل البيانات في قاعدة MySQL. لذلك تحتاج على السيرفر ثلاثة أشياء فقط:

المستخدمونعبر المتصفّح
⟸
سيرفر ShaabApi.NET 8 — الواجهة + الـ API
⟸
قاعدة MySQLتخزين كل البيانات
الخلاصة: ثبّت MySQL + بيئة تشغيل .NET 8 + انسخ ملفات التطبيق واضبط الإعدادات. هذا كل شيء.

خريطة الخطوات (المسار الكامل)

1
المتطلّبات: ثبّت .NET 8 Runtime و MySQL على السيرفر.
2
قاعدة البيانات: أنشئ القاعدة + مستخدماً لها (الجداول تُنشأ تلقائياً).
3
الملفات: جهّز مجلد التطبيق المنشور (publish).
4
الإعدادات: اضبط متغيّرات البيئة (الأسرار + الاتصال + العنوان). ⚠️ أهم خطوة.
5
تشغيل تجريبي: شغّل وتأكّد أن الموقع يفتح والدخول يعمل.
6
تشغيل دائم: كخدمة Windows أو Docker (يعمل تلقائياً عند الإقلاع).
7
الشبكة: افتح المنفذ + اضبط ALLOWED_ORIGINS + (اختياري) HTTPS.
8
التحقق النهائي + النسخ الاحتياطي.

1) متطلّبات السيرفر — ثبّت هذه أولاً

المكوّنالتفاصيل والشرح
نظام التشغيلWindows Server (مفضّل لبيئتكم) أو Linux. أي جهاز ثابت يبقى يعمل.
.NET 8 ASP.NET Core Runtimeهو «المحرّك» الذي يشغّل التطبيق. حمّله من dotnet.microsoft.com/download/dotnet/8.0 واختر ASP.NET Core Runtime 8 (لا تحتاج SDK إلا إن أردت بناء الكود على السيرفر). تحقّق بعد التثبيت بالأمر dotnet --info.
MySQL Server 8.xقاعدة البيانات. حمّلها من dev.mysql.com/downloads (أو MariaDB 10.5+). تأكّد أن خدمتها تعمل بعد التثبيت.
Node.js 18+ (اختياري)فقط إن أردت تشغيل خدمة واتساب المنفصلة. ليست ضرورية لعمل الموقع.
بديل أسهل: إن كان لديك Docker فلا تحتاج تثبيت .NET يدوياً — انظر القسم 6 (ب).

2) إعداد قاعدة البيانات ⚠️

نقطة مهمة ومريحة: التطبيق يُنشئ كل الجداول تلقائياً عند أول تشغيل. أنت فقط تُنشئ القاعدة ومستخدماً لها. افتح MySQL ونفّذ:

CREATE DATABASE shaab_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'shaab'@'localhost' IDENTIFIED BY 'ضع_كلمة_مرور_قوية_هنا';
GRANT ALL PRIVILEGES ON shaab_db.* TO 'shaab'@'localhost';
FLUSH PRIVILEGES;
احفظ كلمة مرور القاعدة — ستحتاجها في القسم 4. ولا حاجة لتشغيل أي ملف SQL يدوياً؛ التطبيق ينشئ كل جدول بنفسه عند الإقلاع.

3) تجهيز ملفات التطبيق

أ) إن كان لديك الكود المصدري + .NET SDK

cd مسار-المشروع
dotnet publish ShaabApi.csproj -c Release -o C:\ShaabApp

ينتج مجلد C:\ShaabApp يحوي ShaabApi.dll مع كل ملفات الموقع (index.html, js, css...). هذا المجلد هو كل ما تنشره.

ب) إن استلمت مجلداً منشوراً جاهزاً

ضعه مباشرةً في مكان ثابت مثل C:\ShaabApp.

4) ضبط الأسرار ومتغيّرات البيئة ⚠️ (أهم قسم)

التطبيق يقرأ إعداداته من متغيّرات البيئة (Environment Variables) — وليس من داخل الكود. هذه القيم إلزامية؛ بدونها يرفض التطبيق العمل (هذا مقصود لأسباب أمنية).

المتغيّرالقيمة / الشرح
ConnectionStrings__DefaultConnectionServer=localhost;Port=3306;Database=shaab_db;User=shaab;Password=كلمة_مرور_القاعدة;CharSet=utf8mb4;
SuperAdminPasswordكلمة دخول «مدير النظام» (السوبر أدمن) — تُحدّدها الإدارة سرّاً وتضعها هنا. غير مدوّنة في هذا الدليل.
AdminPasswordHash8e5fe6d0...c3efbf5 (موجود كاملاً في ملف .env).
Jwt__Keyسرّ عشوائي طويل (مولّد جاهز في .env) — يوقّع جلسات الدخول.
Jwt__IssuerShaabApi
SseTokenسرّ عشوائي (في .env).
ALLOWED_ORIGINS ⚠️عنوان الموقع الذي يكتبه المستخدمون، مثل http://192.168.1.50:8080
PORT8080 (أو أي منفذ تريده).
ملف .env المرفق يحتوي معظم هذه القيم جاهزة. للتوليد من الصفر بدلاً منه: شغّل setup-local-server.ps1.

كيف تضبطها فعلياً؟ — طريقة 1: متغيّرات نظام دائمة (PowerShell كمسؤول)

[Environment]::SetEnvironmentVariable('ConnectionStrings__DefaultConnection',
  'Server=localhost;Port=3306;Database=shaab_db;User=shaab;Password=PASS;CharSet=utf8mb4;','Machine')
[Environment]::SetEnvironmentVariable('SuperAdminPassword','<كلمة-السوبر-أدمن-من-الإدارة>','Machine')
[Environment]::SetEnvironmentVariable('AdminPasswordHash','8e5fe6d0...c3efbf5','Machine')
[Environment]::SetEnvironmentVariable('Jwt__Key','<انسخه من .env>','Machine')
[Environment]::SetEnvironmentVariable('SseToken','<انسخه من .env>','Machine')
[Environment]::SetEnvironmentVariable('ALLOWED_ORIGINS','http://192.168.1.50:8080','Machine')
[Environment]::SetEnvironmentVariable('PORT','8080','Machine')

ثم أعد فتح نافذة PowerShell لتُحمَّل المتغيّرات.

طريقة 2: تحميل من ملف .env قبل التشغيل

Get-Content C:\ShaabApp\.env | ForEach-Object {
  if ($_ -and $_ -notmatch '^\s*#') {
    $k,$v = $_ -split '=',2
    [Environment]::SetEnvironmentVariable($k.Trim(), $v.Trim())
  }
}
احمِ ملف .env: ضعه خارج أي مجلد يُخدَم للويب، وقيّد صلاحياته، ولا ترفعه لأي مكان (git/بريد).

5) تشغيل تجريبي أول (للتأكد)

cd C:\ShaabApp
dotnet ShaabApi.dll

يجب أن ترى سطراً مثل Now listening on: http://0.0.0.0:8080 ورسائل إنشاء الجداول. افتح المتصفّح على http://localhost:8080 — يجب أن تظهر شاشة الدخول. أوقفه بـ Ctrl+C ثم انتقل للتشغيل الدائم.

ظهر خطأ اتصال بقاعدة البيانات؟ راجع سلسلة الاتصال وأن خدمة MySQL تعمل.
شاشة بيضاء أو الدخول لا يقبل السوبر أدمن؟ غالباً المتغيّرات لم تُحمَّل — راجع القسم 4.

6) التشغيل الدائم (اختر واحدة)

أ) كخدمة Windows (موصى به) — عبر NSSM

nssm install ShaabApi "C:\Program Files\dotnet\dotnet.exe" "C:\ShaabApp\ShaabApi.dll"
nssm set ShaabApi AppDirectory C:\ShaabApp
nssm start ShaabApi

المتغيّرات المضبوطة بنطاق «Machine» في القسم 4 تُورَّث للخدمة تلقائياً. الخدمة تعمل عند إقلاع الجهاز وتتعافى عند السقوط.

ب) عبر Docker (بديل)

docker build -t shaabapi .
docker run -d --name shaabapi --env-file .env -p 8080:8080 --restart unless-stopped shaabapi
داخل Docker اجعل Server= في سلسلة الاتصال يشير لعنوان MySQL الفعلي (ليس localhost إن كانت القاعدة خارج الحاوية).

7) الشبكة والوصول ⚠️

  1. افتح المنفذ في جدار الحماية: 
    New-NetFirewallRule -DisplayName "ShaabApi" -Direction Inbound -LocalPort 8080 -Protocol TCP -Action Allow
  2. عنوان ثابت: أعطِ السيرفر IP ثابتاً (مثل 192.168.1.50) أو اسم مضيف داخلي.
  3. ⚠️ ALLOWED_ORIGINS يجب أن يطابق تماماً ما يكتبه المستخدم (البروتوكول + العنوان + المنفذ). إن دخلوا عبر http://192.168.1.50:8080 فاجعلها كذلك بالضبط، وإلا تُرفض الطلبات (خطأ CORS).
  4. (اختياري) HTTPS: ضع التطبيق خلف IIS/Nginx كوسيط عكسي بشهادة داخلية ووجّهه إلى http://localhost:8080.

8) التحقق النهائي بعد التشغيل

  • ☑ الموقع يفتح من جهاز آخر على الشبكة: http://192.168.1.50:8080
  • ☑ دخول مدير الكول سنتر بالرقم الوظيفي 0799 — ومنه تُنشأ بقية الحسابات
  • ☑ دخول السوبر أدمن بالكلمة التي حدّدتها الإدارة
  • ☑ إنشاء سجل تجريبي ثم تحديث الصفحة → السجل محفوظ (يعني القاعدة تعمل)
  • ☑ لوحة السوبر أدمن تفتح وتعرض البيانات

9) الإشعارات (Firebase) — اختياري

الإشعارات الفورية تحتاج مفتاح خدمة Firebase. إن لم تكن مطلوبة داخلياً تجاهل هذا القسم — كل شيء آخر يعمل بدونها. لتفعيلها لاحقاً يُخزَّن المفتاح في قاعدة البيانات تحت المفتاح Shaab_Firebase_Creds (راجِع المطوّر).

10) النسخ الاحتياطي ⚠️ (لا تتجاهله)

اضبط نسخاً يومياً لقاعدة البيانات:

mysqldump -u shaab -p shaab_db > C:\Backups\shaab_20260621.sql

أنشئ مهمة مجدوَلة (Task Scheduler) تشغّلها يومياً. احتفظ بآخر 14–30 نسخة، ويُفضّل نسخة على جهاز آخر. جرّب الاستعادة مرة للتأكد أن النسخ سليمة.

11) الصيانة والتحديث

  • السجلّات: تظهر في مخرجات الخدمة/الحاوية (يمكن توجيهها لملف عبر NSSM).
  • إعادة التشغيل: nssm restart ShaabApi أو docker restart shaabapi.
  • تحديث الموقع: انشر نسخة جديدة فوق C:\ShaabApp ثم أعد تشغيل الخدمة. ترقيات قاعدة البيانات تتم تلقائياً عند الإقلاع.

12) قائمة الأمان النهائية ⚠️

  • ☑ كلمة مرور السوبر أدمن حدّدتها الإدارة سرّاً (غير مدوّنة في أي وثيقة تُسلَّم للفنّي).
  • ☑ ملف .env محمي، غير مخدوم للويب، وغير مرفوع لـ git.
  • ☑ كلمة مرور قاعدة البيانات قوية وغير افتراضية.
  • ☑ جدار الحماية يسمح بالمنفذ المطلوب فقط.
  • ☑ (اختياري) HTTPS عبر وسيط عكسي.
  • ☑ (اختياري) مسح الأسرار القديمة من تاريخ git — راجِع SECURITY_HISTORY_SCRUB.md.
  • ☑ النسخ الاحتياطي اليومي يعمل ومُختبَر.
