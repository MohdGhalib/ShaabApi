# تسليم نظام النسخ الاحتياطي — للمبرمج

هذه الحزمة جاهزة للربط بالسيرفر الداخلي. **كل ما تحتاجه: ملء ملف إعداد واحد + جدولة مهمّة.**

---

## ما هذه الحزمة
نسخ احتياطي على مستوى MySQL لقاعدة بيانات نظام «الشعب»، **يشمل جدول الصور `files` (البايتات)** والعربية — مستقل تماماً عن التطبيق ومتصفّح المستخدم.

| الملف | الدور |
|---|---|
| **`backup-config.ps1`** | ⬅️ **الملف الوحيد الذي تعدّله** (اتصال القاعدة + المسارات) |
| `shaab-backup.ps1` | سكربت النسخ (لا تعدّله — يقرأ الإعداد) |
| `shaab-restore.ps1` | سكربت الاستعادة |
| `README.md` | تفاصيل إضافية وأوامر الجدولة |

---

## الخطوات (بالترتيب)

### 1) المتطلّبات
- أدوات MySQL على السيرفر: `mysqldump.exe` و `mysql.exe` (تأتي مع MySQL Server أو MySQL Client).

### 2) انسخ مجلد `backup/` كاملاً إلى السيرفر
مثلاً إلى `C:\ShaabApp\backup\`.

### 3) عدّل `backup-config.ps1` فقط — املأ:
- `$DbHost` / `$DbPort` / `$DbName` / `$DbUser` / `$DbPassword` — اتصال القاعدة الداخلية.
- `$MysqlBinDir` — مجلد bin لأدوات MySQL (اتركه `''` لو ضمن PATH).
- `$BackupDir` — مجلد النسخ الأساسي على السيرفر.
- `$OffsiteDir` — مجلد خارج الموقع (قرص خارجي/NAS) — اتركه `''` لتعطيله مؤقتاً.
- `$RetentionDays` — مدة الاحتفاظ.

> 🔒 الملف يحوي كلمة مرور القاعدة → احمِ مجلد `backup/` بصلاحيات NTFS (مسؤول/SYSTEM فقط).

### 4) اختبار يدوي
```powershell
powershell -ExecutionPolicy Bypass -File "C:\ShaabApp\backup\shaab-backup.ps1"
```
تحقّق: ظهور `shaab_db_<التاريخ>.zip` في `$BackupDir`، وسطر `✅ اكتمل النسخ` في `backup.log`.

### 5) الجدولة (Task Scheduler) — شغّل كمسؤول:
```powershell
schtasks /Create /SC DAILY /ST 03:00 /TN "ShaabDB Backup Daily" /RU SYSTEM ^
  /TR "powershell -ExecutionPolicy Bypass -NoProfile -File \"C:\ShaabApp\backup\shaab-backup.ps1\""

schtasks /Create /SC HOURLY /TN "ShaabDB Backup Hourly" /RU SYSTEM ^
  /TR "powershell -ExecutionPolicy Bypass -NoProfile -File \"C:\ShaabApp\backup\shaab-backup.ps1\""
```

### 6) اختبر الاستعادة (مهم — مرّة واحدة على قاعدة تجريبية)
```powershell
powershell -ExecutionPolicy Bypass -File "C:\ShaabApp\backup\shaab-restore.ps1" -ZipPath "C:\ShaabBackups\shaab_db_....zip"
```
> غيّر `$DbName` في الإعداد مؤقتاً إلى قاعدة تجريبية للتأكد دون المساس بالإنتاج.

### 7) فعّل «خارج الموقع»
اضبط `$OffsiteDir` على قرص خارجي/NAS يُفصل أو يُزامَن خارج السيرفر (حماية من حريق/سرقة/تلف).

## ما الذي يُنسخ
كل الجداول: `storage` · `montasiat` · `inquiries` · `complaints` · `messages` · `audit_log` · `employees` · **`files` (الصور)** · الإجراءات/المشغّلات.

## تحقّق سريع لاحقاً
```powershell
Get-Content C:\ShaabBackups\backup.log -Tail 20
Get-ChildItem C:\ShaabBackups\shaab_db_*.zip | Sort-Object LastWriteTime -Descending | Select-Object -First 5
```

## ملاحظات
- التطبيق يقرأ اتصال القاعدة من متغيّرات بيئة `MYSQL_HOST/MYSQLDATABASE/MYSQLUSER/MYSQLPASSWORD` — السكربت يقبل نفسها تلقائياً لو تُركت قيم الإعداد على الافتراضي، لكن يُفضّل ضبطها صراحةً في `backup-config.ps1`.
- الباكب داخل التطبيق (لقطات المدير) يبقى كطبقة راحة سريعة فقط — **الأساس هو هذا**.
